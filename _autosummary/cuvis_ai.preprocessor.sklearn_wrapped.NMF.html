fit_transform(X, y=None, W=None, H=None)[source]

Learn a NMF model for the data X and returns the transformed data.

This is more efficient than calling fit followed by transform.

Parameters:

  • X ({array-like, sparse matrix} of shape (n_samples, n_features)) – Training vector, where n_samples is the number of samples and n_features is the number of features.

  • y (Ignored) – Not used, present for API consistency by convention.

  • W (array-like of shape (n_samples, n_components), default=None) – If init=’custom’, it is used as initial guess for the solution. If None, uses the initialisation method specified in init.

  • H (array-like of shape (n_components, n_features), default=None) – If init=’custom’, it is used as initial guess for the solution. If None, uses the initialisation method specified in init.

Returns:

W – Transformed data.

Return type:

ndarray of shape (n_samples, n_components)

get_feature_names_out(input_features=None)

Get output feature names for transformation.

The feature names out will prefixed by the lowercased class name. For example, if the transformer outputs 3 features, then the feature names out are: [“class_name0”, “class_name1”, “class_name2”].

Parameters:
